## 二、严重问题（阻断性问题）

### S-001: 敏感文件泄露风险

| 项目 | 内容 |
|------|------|
| **问题描述** | `camforge-next.keystore` 文件位于项目根目录，可能被意外提交到 Git 仓库。keystore 文件包含 Android 应用签名密钥，一旦泄露可能导致应用被恶意篡改后重新签名发布。 |
| **问题位置** | `camforge-next.keystore` (项目根目录) |
| **严重程度** | 🔴 严重 |
| **问题类别** | 安全 |
| **影响范围** | Android 应用发布安全 |
| **修复建议** | 1. 立即将 keystore 文件添加到 `.gitignore`；2. 如果已提交，立即从 Git 历史中移除并轮换密钥；3. 仅通过 GitHub Secrets 存储 Base64 编码的 keystore |

### M-001-B: TIFF 导出功能未实现

| 项目 | 内容 |
|------|------|
| **问题描述** | 导出界面中，快速导出的图片导出声称默认导出 600DPI 的 TIFF 图像，但实际导出的是 PNG 格式。自定义导出处即使选择了 TIFF 格式，导出的也是 PNG 格式。`generateTIFF` 函数仅是 `generateHighResPNG` 的别名，并未实现真正的 TIFF 编码。TIFF 格式对于工程制图和专业打印有重要意义，需要实现真正的 TIFF 导出功能。 |
